# MCP Admission-Chain Surface

**Legend:** DELIVERY

Promote the agent surface from "CLI JSON plus intention" to an explicit,
read-only MCP delivery adapter over debugger readings and admission-chain
posture.

## Origin

The ontology catch-up clarified that WARP TTD observes and explains the lawful
optic chain. It does not compile artifacts, issue authority, admit runtime
execution, or mutate local strands.

That makes MCP the next agent-facing surface, but only under a narrow first
contract:

- read session, worldline, and reading facts through `DebuggerSession`
- expose adapter capabilities as support facts
- expose admission-chain facts as present, absent, or obstructed posture
- do not issue grants
- do not perform admission
- do not create strands
- do not mutate host state
